# 이 솔루션 안내

clone 받은 팀원이나 처음 보는 사람에게 이 솔루션을 설명하는 스킬이다. 비개발자 기준으로, 전문용어는 한 줄로 푼다.

## 하는 일
1. `axfm.json`(이름·설명·제공/수신), `README.md`, `axfm/interface.md` 를 읽는다.
2. 다음 4가지를 쉬운 말로 요약한다:
   - **무엇**: 이 솔루션이 하는 일 한 문장
   - **실행**: `.\start.ps1` (막히면 `.\start.cmd`) — 화면 없는 자동화 스크립트임을 알려준다
   - **주고받는 것**: provides/accepts 목록과 의미 (상세는 axfm/interface.md)
   - **다음 할 일**: 개발을 이어가려면 `/axfm-guide`
3. 파일이 없거나 읽기 실패면 원인을 추측하지 말고 `/axfm-guide` 로 안내한다.

## 규칙
- 이 스킬은 이 솔루션 폴더 안에서만 동작한다 (AXFM 워크스페이스 규칙: 폴더에 넣으면 그 폴더에서만 적용).
- 3~7줄로 짧게. 질문받지 않은 내용을 늘어놓지 않는다.
