# AXFM 새 솔루션 만들기

## 원칙
- **직접 템플릿을 복사하거나 플레이스홀더를 치환하지 않는다.** 반드시 `scaffold.mjs`를 호출한다 (결정적·검증 포함).
- 질문은 최대 3개. 사용자는 비개발자일 수 있다 — 전문용어는 한 줄로 풀어 말한다.

## 1단계: 질문 3개
1. **종류**: 웹앱(Next.js, 화면 있음) / 자동화 스크립트(Python, 화면 없음) 중 하나.
2. **한 줄 설명**: 팀원이 목록에서 볼 설명.
3. **이름**: 한글 가능. (id는 자동으로 ASCII 슬러그로 만든다 — 사용자에게 물어보지 말 것)

만든 사람 이름은 사용자 환경에서 자동 사용(없으면 물어본다). id 슬러그는 이름을 로마자화하지 말고,
사용자에게 `{owner}-{간단한영문}` 형태를 제안해 확인받는다 (예: 점심추천기 → `hong-lunch`). id는 소문자·숫자·하이픈만.

## 2단계: 환경 빠른 점검
- 종류가 웹앱이면 `node --version`(20+), 스크립트면 `python --version`(3.10+) 확인.
- 없으면 설치 안내 후 중단 (Node: `winget install OpenJS.NodeJS.LTS`, Python: https://www.python.org).
- 대상 폴더 경로를 정한다(기본: 현재 작업 폴더 하위 `<id>`). **경로는 ASCII 권장**(한글 경로는 일부 도구에서 문제).

## 3단계: 생성 (결정적 스크립트 호출)
```
node "${CLAUDE_PLUGIN_ROOT}/scripts/scaffold.mjs" --type <nextjs|python> --id <id> --name "<이름>" --desc "<설명>" --owner "<만든이>" --dest "<대상경로>"
```
- 스크립트가 병합 복사·치환·(웹앱)디자인 토큰 내보내기·레지스트리 등록·사후검증까지 수행한다.
- 출력 JSON의 `ok: true`와 `port`(웹앱)를 확인한다. 실패하면 오류 메시지를 그대로 사용자에게 보여주고 원인(대상 폴더 비어있지 않음 / id 충돌 등)을 설명한다.

## 4단계: 첫 실행 안내 (딱 이 두 가지만 — 명령은 한 줄씩, `&&` 금지: 기본 PowerShell 5.1 비호환)
- 웹앱: `cd <대상>` → `npm install` → `npm run dev` → "브라우저에서 http://localhost:<port> 를 여세요" (port 는 scaffold 출력값)
  - npm 설치가 사내망에서 막히면 `/axfm-guide`가 진단하도록 안내.
- 스크립트: `cd <대상>` → `.\start.ps1` (막히면 `.\start.cmd`)

## 5단계: 진행도 + 추천 스킬
- 대상 폴더에 `.axfm/progress.json`이 없으면 `{"milestones":{"created":true}}`로 만든다.
- 스택별 추천 외부 스킬 설치를 **제안만** 한다(강제 아님): `docs/recommended-skills.md` 참조.
- 마무리 문구는 항상 다음 한 줄로 끝낸다: "개발을 시작하려면 이 폴더에서 claude 를 실행하고 /axfm-guide 를 입력하세요."
