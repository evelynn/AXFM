# 솔루션 워크스페이스 커스터마이징 프레임워크 — 검토 평가·보완 계획 (2026-07-03)

> 오너 제안: 솔루션에 SKILL·MCP·TOOL·PLAN·DOC·DESIGN 등 **명시적 폴더**를 두고, 그 위치의 내용으로
> 업그레이드·변경이 가능하며, **해당 폴더 안에서만 적용**(그 폴더에서 실행하는 AI가 인식·적용)되게 한다.
> 설치가 필요한 스킬(/qa 류)은 폴더에 넣어두고 최초 구동 시 설치되게 한다.
> 본 문서 = 이 제안의 타당성 평가 + 부족한 점 + 실행 계획. (CC 공식 문서 실사 2026-07-03, CC 2.1.198 기준)

## 1. 평가 — **타당하다.** CC 네이티브 폴더 스코프 메커니즘과 1:1 매핑

제안한 폴더들이 Claude Code가 이미 폴더 단위로 인식하는 위치와 그대로 대응된다.
프레임워크가 발명할 것은 "표준 배치 + 부트스트랩 + 규칙"뿐이며, 인식 엔진은 CC가 제공한다.

| 제안 폴더 | CC 네이티브 실체 | 그 폴더에서의 적용 방식 | 신뢰 게이트 |
|---|---|---|---|
| **SKILL** | `.claude/skills/<name>/SKILL.md` | **자동 로드 — 설치 단계 자체가 없음.** 파일이 곧 스킬, 폴더 밖에선 무효 | 없음 (즉시) |
| **MCP** | `.mcp.json` (솔루션 루트) | 자동 감지, 승인된 서버만 연결. `enabledMcpjsonServers`로 사전승인 가능 | 워크스페이스 신뢰 대화상자 |
| **TOOL** | `tools/`(규약) + `.claude/agents/` | 결정적 스크립트는 스킬·CLAUDE.md가 호출. 서브에이전트는 자동 발견 | 에이전트: 없음 |
| **PLAN** | `docs/plan/` (규약) | 자동 아님 — CLAUDE.md 규칙 + 스킬(idea/feature)의 표준 위치 지정으로 "적용"을 정의 | — |
| **DOC** | `docs/` (규약) | 〃 | — |
| **DESIGN** | `design/` (규약) | 〃 + 프레임워크 표준 상속 규칙 | — |
| 규칙 선언부 | `CLAUDE.md` | 자동 읽음 — **폴더 맵(무엇이 어디에, 무슨 역할)을 선언하는 곳** | 없음 |
| 활성화 | `.claude/settings.json` | 폴더 스코프 활성화(enabledPlugins 등) — v0.2.0에서 이미 사용 중 | 신뢰 승인 후 적용 |

**핵심 발견 2가지 (설계를 바꾸는 사실):**
1. **동봉 = 설치다.** 프로젝트 로컬 스킬은 신뢰 프롬프트도 설치 명령도 없이 그 폴더에서 자동 적용된다.
   → "/qa 같은 스킬"은 템플릿에 파일로 동봉하면 끝 — "최초 구동 시 설치"가 필요 없다.
2. **외부 플러그인은 자동 설치가 불가능하다** (`enabledPlugins`는 활성화만, 설치는 항상 명시적).
   → "최초 구동 시 설치"는 자동이 아니라 **부트스트랩 안내 설치**(점검 → 목록 제시 → 확인 → 설치 실행)로 설계해야 한다.

기존 원칙과의 정합: DEC-9(폴더 스코프)의 자연스러운 확장이며, 과제 ②(환경·스킬 세팅)와 ⑤(개발 관리)에 직접 기여.
clone 받은 팀원의 AI도 같은 규칙·도구·계획 문서로 작동하게 되어 "솔루션 = 자기 기술적(self-describing) 작업공간"이 된다.

## 2. 표준 워크스페이스 레이아웃 v1 (안)

```
<솔루션>/
  CLAUDE.md                  ← AI 규칙 + 폴더 맵 선언 (자동 읽음, 200줄 이내 유지)
  axfm.json                  ← 정체성 (+ requires: 외부 플러그인 의존성 선언 — 신규 필드)
  .claude/
    settings.json            ← 폴더 스코프 활성화 (기존)
    skills/                  ← [SKILL] 이 솔루션 전용 스킬 — 파일 = 즉시 적용
    agents/                  ← [TOOL·AI] 전용 서브에이전트 (선택)
  .mcp.json                  ← [MCP] 이 솔루션 전용 MCP 서버 (선택, 신뢰 승인 후)
  tools/                     ← [TOOL] 결정적 스크립트 (스킬이 호출)
  docs/plan/                 ← [PLAN] 기획·계획 (idea/feature 스킬의 표준 읽기·쓰기 위치)
  docs/                      ← [DOC] 문서
  design/                    ← [DESIGN] 로컬 디자인 — 추가 토큰만 (표준 v1.0 상속)
  axfm/interface.md          ← 연동 계약 (기존)
```

### 설계 결정 (제안)
- **W-D1 폴더명은 CC 네이티브 경로 그대로.** 별도의 대문자 폴더(SKILL/ 등)를 발명하지 않는다 —
  CC가 인식하는 위치는 정해져 있고, 별칭 폴더는 동기화 드리프트만 낳는다. 한국어 역할 맵은 CLAUDE.md에 표로 선언.
- **W-D2 설치 2계층.** ① 동봉물(로컬 스킬·에이전트·mcp.json) = 파일이 곧 적용, 설치 없음.
  ② 외부 플러그인 = `axfm.json`의 `requires` 배열로 선언 → 부트스트랩이 점검·안내 설치.
- **W-D3 부트스트랩 = `setup.mjs`(결정적) + /axfm-guide 통합.** 최초 구동 시: requires 대조 → 미설치
  목록·출처 제시 → 사용자 확인 → `claude plugin install <p> --scope project` 실행. 자동 무확인 설치 금지.
- **W-D4 DESIGN 로컬은 "추가"만.** 표준 토큰(색·타이포) 오버라이드는 기본 금지(명시 옵트인 + /axfm-qa 경고) — 과제 ④ 보호.
- **W-D5 레이아웃도 버전 관리.** `axfm.json`에 `layout` 버전 필드, sync-modules가 레이아웃 업그레이드(폴더 추가·CLAUDE.md 맵 갱신)를 포함.
- **W-D6 이름 규약.** 우선순위가 개인(~/.claude/skills) > 프로젝트 > 플러그인이므로, 솔루션 동봉 스킬은
  범용 이름(/qa) 대신 충돌 안전 이름(예: /qa-check 또는 `<id>-qa`)을 권장.

## 3. 부족한 점·리스크와 보완 (검토에서 도출)

| # | 리스크 | 보완 |
|---|---|---|
| R1 | **보안 — clone = 타인의 지시문·서버를 내 AI에 주입.** 로컬 스킬은 신뢰 프롬프트 없이 로드됨(실사 확인). .mcp.json은 신뢰 게이트가 있으나 광범위 사전승인 키가 존재 | clone 등록 흐름(guide)에 **동봉물 고지 단계** 필수: setup.mjs가 동봉 스킬·에이전트·MCP 목록을 결정적으로 출력하고 사용자 확인 후 진행. `enableAllProjectMcpServers` 사용 금지 규칙. 외부 requires는 사내 allowlist 검사(서비스화 계획 2-5와 합류) |
| R2 | "최초 구동 시 설치"가 CC 제약상 자동 불가 | W-D3의 안내 설치로 설계하고 문서 약속도 "확인 후 설치"로 정직하게 |
| R3 | PLAN/DOC/DESIGN은 CC가 자동 적용하는 폴더가 아님 — 폴더만 만들면 장식이 됨 | CLAUDE.md 규칙("화면 작업 전 design/ 확인" 등) + 스킬 개정(feature가 docs/plan 확인, idea가 docs/plan에 저장)으로 "적용"을 행동으로 정의 |
| R4 | 개인 스킬이 프로젝트 스킬을 덮는 우선순위 → 조용한 오버라이드 혼란 | W-D6 이름 규약 + /axfm-qa에 스킬 섀도잉 점검 항목 추가 |
| R5 | 동봉물의 업그레이드 전파 — 프레임워크 소유 vs 사용자 소유 경계가 폴더 단위로 흐려짐 | 소유권을 CLAUDE.md 폴더 맵에 명시(예: 동봉 기본 스킬은 프레임워크 소유·sync 대상, 사용자 추가 스킬은 불가침). sync-modules 전파 목록에 동봉 기본 스킬 포함 |
| R6 | 검증 공백 — 동봉 스킬·레이아웃이 게이트 밖 | scaffold 사후검증에 레이아웃 검사 추가, test-skills 린트(서비스화 계획 1-4)의 대상에 템플릿 동봉 스킬 포함, setup.mjs 게이트 신설 |
| R7 | CLAUDE.md 비대화(200줄 지침) | 폴더 맵은 표 1개로 제한, 상세는 docs/workspace.md로 분리 |
