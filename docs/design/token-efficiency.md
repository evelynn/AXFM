# 토큰 효율 연구 (2026-07-05, 설계 4원칙 "알뜰"의 근거 문서)

> 배경: 일반 사용자의 요금제는 **Pro 가 기본**이다 — 스킬은 "기본적인 도움"을 주되 사용자의 토큰 한도를
> 아껴야 한다. 이 문서는 비용 구조의 측정, 이미 적용된 절약 장치, 신규 규칙, 향후 연구 항목을 관리한다.
> 재측정 명령: `claude plugin details axfm@axfm` (릴리스마다 아래 표 갱신).

## 1. 비용 구조 — 두 종류의 비용

| 종류 | 언제 드나 | 관리 방법 |
|---|---|---|
| **상시(always-on)** | 활성화된 폴더에서 **모든 세션마다** — 스킬 description 이 컨텍스트에 실림 | description 을 짧게, 스킬 수를 최소로, 폴더 스코프로 무관 세션 비용 0 |
| **호출 시(on-invoke)** | 스킬이 실제로 발동할 때 — SKILL.md 본문 로드 + 스킬이 읽는 파일들 | SKILL.md 를 짧게, 상세는 docs 로 분리, 필요한 파일만 읽기 |

## 2. 실측 (v0.5.0 기준, plugin details)

- **상시 합계: ~1,132 tok/세션** (v0.2.0 9종 1,266 → 8종 정리로 -134)
  — Pro 대략치로 하루 수십 세션을 열어도 상시 비용은 체감 미미한 수준. 관리 목표: **1,500 미만 유지**.

| 스킬 | 상시 | 호출 시 | 비고 |
|---|---|---|---|
| guide | 160 | ~1.9k | 진입점 — 가장 자주 호출되므로 본문 증가에 가장 민감. 새 기능은 본문이 아니라 스크립트로 |
| connect | 180 | ~1.5k | |
| new | 160 | ~1.3k | |
| feature | 150 | ~1.1k | |
| publish | 110 | ~1.1k | |
| debug | 120 | ~0.7k | |
| qa | 110 | ~0.54k | |

- 진짜 비용은 표 밖에 있다: **스킬이 발동한 뒤 읽는 파일과 재시도**가 SKILL.md 본문보다 훨씬 크다.
  그래서 절약의 핵심은 본문 압축이 아니라 **"LLM 이 하는 일 자체를 줄이는 구조"**다.

## 3. 이미 적용된 구조적 절약 장치

1. **결정적 스크립트 위임** — 생성(scaffold)·등록(register)·동기화(sync)·점검(publish-check/check-env)을
   LLM 파일 편집이 아니라 스크립트 1회 호출로: 도구 호출 수·재시도·출력 토큰을 구조적으로 절감. (최대 장치)
2. **폴더 스코프** — 활성화 폴더 밖에서는 상시 비용 0.
3. **스킬 8종 정리** — 중복(toolbox) 제거, guide 가 흡수.
4. **응답 형태 규칙** — "다음 행동 딱 하나", "질문 최대 N개", "3~7줄", "이모지 최대 1개" 가 스킬마다 내장.
5. **집계는 코드가** — `overview()` 공통 함수가 전 솔루션 현황을 계산해 반환, LLM 은 요약만 (2026-07-05 신설).
6. **문서 분리** — SKILL.md 는 절차만, 상세(추천 목록·증상표)는 docs 로 — 필요할 때만 읽힘.

## 4. 신규 규칙 (2026-07-05 적용)

- 스킬·CLAUDE.md 공통: **필요한 파일만 읽는다** (전체 재읽기 금지), 반복 집계·검사는 스크립트/공통 함수에 위임.
- guide: 새 책임은 본문 추가가 아니라 **스크립트 신설 + 호출 한 줄**로 (본문 1.9k 상한 관리).
- 릴리스 절차에 토큰 재측정 포함 — 상시 합계가 1,500 tok 을 넘으면 스킬 축소/문서 분리 검토.

## 5. 향후 연구 항목

- **파일럿에서 실사용 측정**: 참가자별 세션당 토큰 사용량(가능하면 /usage 자가 보고)과 스킬 발동 횟수 —
  "설계상 추정"을 실측으로 교체.
- description 다이어트: 상시 비용의 대부분은 description — 트리거 문구를 유지하며 압축 가능한지 A/B.
- `claude plugin eval` 활용: 스킬별 golden 시나리오의 토큰 소비 비교 (축소·통합 판단 자료).
- idea(0.48k)·qa(0.54k) 통합 시 상시 -260 tok 추가 절감 — 파일럿 사용 빈도와 함께 판단 (DEC-7).
