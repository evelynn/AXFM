Hansol AXFM

v0.2.0 디자인 표준 v1.0

바이브 코딩을 위한 Claude Code 플러그인 프레임워크 — 개발이 처음이어도 괜찮습니다.
만들기(/axfm-new) · 연동하기(/axfm-connect) · 배우기(/axfm-guide)를 한 벌로 제공합니다.

1. 플러그인 설치 — 해당 폴더에만 적용

준비물: Claude Code(로그인 완료) · 웹앱을 만들려면 Node.js 20+ · 자동화 스크립트를 만들려면 Python 3.10+

① 마켓플레이스 등록 (PC당 최초 1회)

claude plugin marketplace add F:\AI_MAKE\hansol-axfm     ← 또는 사내 git 저장소 주소

② 작업 폴더에서 설치

cd <작업 폴더>
claude plugin install axfm@hansol-axfm --scope project
폴더 스코프 — 플러그인은 설치한 폴더에서만 켜집니다. 다른 프로젝트에는 /axfm- 스킬이 나타나지 않아 전역 환경을 어지럽히지 않습니다. (활성화 정보는 그 폴더의 .claude/settings.json 한 곳에만 기록)

확인: 그 폴더에서 claude 실행 후 /axfm- 입력 → 스킬 9개 목록이 뜨면 성공.

2. 첫 솔루션까지 10분 — 전체 흐름

  1. 만들기 — /axfm-new 입력, 질문 3개(종류/설명/이름)에 답하면 뼈대가 자동 생성됩니다. 레지스트리 등록·디자인 적용까지 자동.
  2. 실행해서 눈으로 확인 — 웹앱: npm install && npm run dev → 브라우저 확인 / 스크립트: .\start.ps1 (막히면 .\start.cmd)
  3. 다음 할 일 안내 — 솔루션 폴더에서 claude 실행 → /axfm-guide. 지금 어디까지 왔고 다음에 뭘 할지 딱 하나만 알려줍니다.
  4. 기능 추가 — /axfm-feature 에 원하는 것을 말로 설명하면 작게 만들고 눈으로 확인시켜 줍니다.
  5. 연동·공유 — /axfm-connect 로 다른 솔루션과 데이터 연결, /axfm-publish 로 팀에 공유.

3. 스킬 9종 — 무엇을 해주나

명령언제 쓰나하는 일
/axfm-guide뭘 해야 할지 모를 때 (늘 진입점)프로젝트 상태를 직접 진단해 지금 할 일 딱 하나를 제안. 5단계 여정 게이지, clone 등록, 모듈 업데이트도 담당
/axfm-new새 도구를 만들고 싶을 때질문 3개만 받고 웹앱(Next.js) 또는 자동화 스크립트(Python) 뼈대를 자동 생성 — 등록·디자인·설정 포함
/axfm-idea아이디어가 막연할 때무엇을/누가/왜 세 가지만 물어 한 페이지 기획 메모로 정리, 가장 작은 첫 버전을 함께 정함
/axfm-feature"~하게 해줘", 버튼·기능 추가기능을 작게 구현하고, 연동 계약(interface.md)을 자동으로 함께 갱신, 결과를 눈으로 확인시킴
/axfm-connect다른 솔루션과 데이터 주고받기이웃 솔루션 탐색 → 상대 연동 문서 확인 → 연결 코드(connector) 생성 → 실제로 데이터가 오가는지 확인
/axfm-debug"안 돼", "에러 나"흔한 원인(환경/실행 위치/연동/버전)부터 순서대로, 한 번에 하나씩 진단하고 고침
/axfm-qa공유 전 "잘 되나 봐줘"내 솔루션 맞춤 체크리스트 생성 → 항목별로 실제 동작·디자인·문서 일치를 점검하고 표로 보고
/axfm-publish팀원도 쓰게 하고 싶을 때비밀값·데이터 유출 점검 → git 준비 → 메신저에 붙일 팀원용 안내문 자동 생성
/axfm-toolbox"뭐 깔면 좋아?"검증된 외부 스킬/플러그인 중 내 상황에 맞는 3~5개만 골라 설치법과 함께 추천

4. 표준 연동

솔루션끼리는 서버 없이 표준 스냅샷 파일로 데이터를 주고받습니다. 상대가 꺼져 있어도 마지막 데이터를 읽을 수 있습니다.

  • • 계약서: 각 솔루션의 axfm/interface.md — 이 문서만 보면 연동 가능
  • • 공통 함수: writeShared / readFrom — 모든 솔루션이 같은 함수 사용
  • • 범위: 같은 PC (팀원 솔루션은 clone 받아 내 PC에서 실행하면 이웃이 됨)
  • • 웹앱↔스크립트 교차 연동까지 자동 테스트로 상시 검증

5. 디자인 통일

모든 솔루션이 표준 디자인 v1.0(2026-07-03 확정)을 따릅니다 — 색·간격·글자가 시스템마다 제각각이 되지 않습니다.

  • • 단일 기준: 플러그인의 design/DESIGN.md 토큰
  • • 모든 화면 작업은 토큰만 사용 (임의 색상·간격 금지 규칙 내장)
  • • 디자인이 바뀌면 파일 교체만으로 전 솔루션에 반영 (코드 수정 0)

6. 안심 장치 — 위험한 일은 사람이 아니라 스크립트가

실수하기 쉬운 단계는 매번 똑같이 동작하는 스크립트가 처리하고, 자동 테스트가 상시 검증합니다.

스크립트역할
scaffold.mjs솔루션 생성 — 복사·치환·디자인·등록·사후검증까지 한 번에
register.mjsclone 받은 솔루션을 이 PC 목록에 등록
sync-modules.mjs공통 모듈 업데이트 — 덮어쓰기만, 내 파일은 절대 삭제 안 함
export-design.mjs디자인 토큰을 CSS로 변환
test-all.mjs전체 자동 검증 — 디자인·PowerShell 호환·연동 왕복(파이썬↔파이썬, 웹앱↔파이썬)·단위 테스트

7. 팀원이 내 솔루션을 받는 법

  1. git clone <주소> → cd <폴더>
  2. (최초 1회) claude plugin marketplace add <사내 저장소> → claude plugin install axfm@hansol-axfm --scope project
  3. 실행 — 웹앱 npm install && npm run dev / 스크립트 .\start.ps1
  4. 같은 폴더에서 claude → /axfm-guide — 자동으로 등록되고 연동 준비 완료

솔루션 폴더에는 폴더 스코프 활성화 설정이 이미 들어 있어, Claude Code 가 설치를 스스로 안내합니다.
